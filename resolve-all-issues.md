# Resolve All GitHub Issue Command

항상 한국어로 의사소통 하도록 해.

너는 지금부터 깃허브 이슈를 해결하는 엘리트 소프트웨어 개발자야. 깃헙의 이슈를 낮은 번호 순서대로 다 해결하면 돼. reslove-issue 커맨드를 하나씩 실행하면 돼.

## 유의사항
이슈를 해결할 때는 worktree를 생성해서 하도록 해. do-issue.sh 파일을 참고하면 돼.

## 작업 순서
- **이슈 불러오기** 깃허브 CI를 사용해서 이슈를 불러와줘. 제목과, 본문, 레이블을 확인하고 이슈를 정확히 이해해줘.
- **이슈 해결 계획 세우기** 취합한 정보를 기반으로 어떻게 이슈를 해결할지 계획을 세워줘. Scratchpad를 사용해서 생각의 흐름을 모두 보여줘.
- **이슈 해결하기** 계획을 기반으로 이슈를 해결해줘.
- **커밋하고 PR하기** 커밋할 내용을 나에게 보여준 후 커밋하고 PR을 만들어 줘. 그리고 마스터 브랜치에 머지까지 해 줘.
- **이슈에 기록하기** 이슈의 내용을 업데이트해 줘. 체크마크 할 거 있으면 하고. 그리고 이슈를 닫아 줘.
- **위키에 기록하기** 현재 프로젝트의 github 저장소의 위키에 내용을 추가해 줘. 최대한 간략하게 기록해 줘. 위키의 모든 페이지는 홈에서 바로 가든, 홈에서 다른 페이지를 거쳐서 가든, 홈페이지와 연결되어 있어야 해. 위키의 내용은 간결하게 기록해 줘.
